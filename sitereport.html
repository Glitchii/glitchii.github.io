Introduction

Unlike probably most of my classmates, I've had prior experience (about 2 years) in web development, therefore, I have a good grasp of the basics.

Files & file structure

There are 6 HTML files in this assignment project. All HTML files are in the root directory of the project. I made seperate folders for other files, such as js directory for JavaScript files, CSS for CSS files, imgs for images etc. All non-HTML files/folders are in the assets directory. The index.html holds HTML code for the homepage. Most pages are linked in the header including the contact page and home page & some pages won't be displayed based on the active page. Since this is actually my personal website, I decided not to include some pages eg. sitereport, blog, & portofolio in the the menu since they will be useless to most people coming to view my website.

While every HTML file has its own CSS and (if used) JS files, I also have global files such as index.css and index.js. The code in these files are used by all pages. index.css is imported in all CSS files with CSS's `@import` rule. You can find this line on the first line of all CSS files except the index.css (itself). index.js is imported through all HTML files. index.css has styles for content like the header (which is the same HTML code in all HTML fiels except a few changes in links) which is on all pages and index.js has code like event listeners (etc) for the header and other global elements.

Theme switching & useful information

Pressing the icon in the top right of the website, reveals buttons to switch themes in the bottom right and useful information like when the website was last updated in the bottom right. The date when the website was last updated is not added in manually, instead is retrieved through a get request to GitHubs API on the repo https://api.github.com/repos/Glitchii/glitchii.github.io to get the date of the last commit in the repository. When a theme is selected, a value is stored in the local storage of the browser which is loaded when the website is accessed again to use the user's prefered theme.

Summary

Due to my head-start in in web development, I have had less of a bad experience working completing the assignment.
While JavaScript is not included in the assignment brief, I used it to make the website dynamic.
I used JavaScript in in many parts of the website to:

  • Removing the loading screen on the homepage.
  • Dynamically add projects to the projects list from an object t avoid repeating and updating multiple HTML files when I add a new project.
  • Toggle the 'projects' menu in the header.
  • Toggle the harmburger menu.
  • Toggle & animate the 'contact' section in the homepage when 'contact' is clicked.
  • Display the current URL in the 404 page
  • Display the URL alongside the domain in the main detail tags on the sitereport page
  • Alter the message to add author and email (if provided) at the end on an email when submited in the contact form

The 404 page is also not required, but I have included it to have my own 404 page show when the user goes to a page that does not exists rather than showing GitHub Pages's default 404 page.

Responsiveness

As also required, I made sure to make every single page of the website responsive. This means that the website will work on any device, from a smartphone to a desktop and most browser, either Chromium or Gecko based. Probably the only exception to browsers is Internet Explorer as is a very old and unsuppoted browser, and I have not been able to test it on my computer. Although, I do not expect anyone exessing this site to be using it.
